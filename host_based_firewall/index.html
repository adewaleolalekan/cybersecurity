🔥 Host-Based Firewall Configuration on Windows

📌 Project Overview

This project demonstrates how to harden a Windows system by configuring host-based firewall rules using Windows Defender Firewall with Advanced Security. The configuration applies Zero Trust principles by whitelisting trusted applications and blocking unverified ones.

🛠️ Tools Used

  • Windows 10 or 11
  • Windows Defender Firewall with Advanced Security
  • Optional: PowerShell or wf.msc

✅ Objectives

  • Backup existing firewall rules
  • Allow only trusted applications
  • Block unsafe protocols (e.g., FTP) and IPs
  • Test and verify rule effectiveness

🧩 Configuration Steps

🔹 Step 1: Access Firewall Interface

Open Windows Defender Firewall with Advanced Security from the Start menu.

🔹 Step 2: Backup Rules

Navigate to Action > Export Policy and save as FirewallBackup.wfw.

🔹 Step 3: Inbound Rules

  • Allow Firefox
    • Program: C:\Program Files(x86)\Mozilla Firefox\firefox.exe
    • Action: Allow
    • Profiles: Domain, Private, Public
  • Block FTP
    • Protocol: TCP
    • Port: 21
    • Action: Block
    • Profiles: All

🔹 Step 4: Outbound Rules

  • Allow Microsoft Edge
    • Program: C:\Program Files(x86)\Microsoft\Edge\Application\msedge.exe
    • Action: Allow
    • Profiles: All
  • Block Facebook IP
    • Remote IP: 102.132.101.35
    • Action: Block
    • Profiles: All
